# コントリビューションガイド

## Issue と PR

- 不具合・要望は **GitHub Issue** で共有してください。
- 修正は **Pull Request** を推奨します。`master`（または既定ブランチ）向けに送ってください。
- CI（`.github/workflows/build.yml`）の **Release ビルド**と **`dotnet test`（`VBASSH.Tests`）** が通ることを確認してください。
- **`gh issue comment` 等に長文を渡すとき:** Windows の **PowerShell はバッククォート（`` ` ``）を解釈**するため、本文が欠けたり化けたりします。UTF-8 の JSON をファイルに書き、**`gh api ... --input`** で渡すか、**cmd.exe** から実行するなどして回避してください。

## COM / VBA 互換（破壊的変更の扱い）

本ライブラリは **Excel VBA 等から COM で呼ばれる**前提です。次を変えると、既存ユーザーに **再コンパイル・再登録・参照し直し**が必要になることがあります。

| 変更の種類 | 影響の目安 |
|------------|------------|
| `ComClass` の **ClassId / InterfaceId / EventsId**（`clsVbaSSH.vb` 内の GUID） | 既存の ProgID/型ライブラリ登録と不整合になりやすい。**やむを得ない場合はリリースノートで明示**し、利用者に **RegAsm のやり直し**を案内する。 |
| `VbaSSH.Open` / `VbaSshLogin` の **プロパティ名・型**、`Execute` / `Close` の **シグネチャ・戻り値型** | VBA の既存コードが **コンパイルエラー**になる可能性が高い。 |
| **依存 SSH.NET のメジャー相当更新** | 動作や例外の出方が変わることがある。リリースノートに記載する。 |

**推奨:** 互換を壊す変更は **セマンティックバージョンのメジャー**（例: アセンブリ `3.0.0.0`）とセットで検討する。

**2.2.0.0:** `VbaSSH.Open` の戻り値は **`String` から `Boolean` に変更**されました。既存 VBA は `If ssh.Open(login) Then` 形式への修正が必要です。失敗理由は **`LastError`**、非永続シェル時の終了コードは **`LastExitStatus`** を参照してください。

## 強名キー（`VBASSHKEY.snk`）

- リポジトリに **公開されている**ため、「秘密の鍵」ではありません（オープンソースで強名を付ける一般的な形です）。
- **鍵を差し替える**とアセンブリ識別が変わり、利用者は **再ビルド・再 RegAsm** が必要です。差し替えはプロジェクト方針で合意のうえ実施してください。
- ローカルでは **リポジトリのクローン＋ Git 管理**が事実上のバックアップになります。社内フォークで鍵だけ変える場合は、配布物との対応関係に注意してください。

## ビルド・配布の注意

- CI および通常のコマンドラインでは **`RegisterForComInterop=false`** でビルドし、COM 登録は **RegAsm を別途**行う運用に合わせています。
- `bin\Release` には **複数の依存 DLL** が出力されます。配布時は **同じフォルダに揃える**必要があります（`README.md` 参照）。

- 既定の **`UsePersistentShell`** は対話シェル経由のため、リモートのシェル・`echo` の前提が変わると挙動に影響します。特殊な環境では `False` に戻すか、コマンドを `&&` で連結する運用を検討してください。
